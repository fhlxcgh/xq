# 中国象棋标准中文棋谱记法扩展设计

## 目标

在现有网页版中国象棋基础上，把走棋历史从简化记法升级为更标准的中文棋谱记法，同时保留当前的 `红/黑` 前缀。

## 输出格式

历史记录仍显示为单条字符串，但记法规则升级为：

- 保留 `红` / `黑` 前缀
- 使用中文数字：`一二三四五六七八九`
- 无冲突时使用常规列位记法
- 有同类棋子冲突时使用 `前` / `后` 消歧

示例：

- `红车九进一`
- `黑马二进三`
- `红前车平五`
- `黑后炮进二`
- `红前兵进一`

## 记法规则

### 1. 基本格式

标准输出结构：

- `<阵营><棋子称谓><起始标识><动作><目标标识>`

例如：

- `红车九进一`

### 2. 棋子称谓

沿用当前界面用字：

- 红方：车、马、相、仕、帅、炮、兵
- 黑方：车、马、象、士、将、炮、卒

### 3. 数字规则

红黑双方统一使用中文数字：

- 一、二、三、四、五、六、七、八、九

不再对黑方使用阿拉伯数字。

### 4. 无冲突时的列位记法

当同类型棋子在本方阵营内不需要消歧时：

- 起始标识用“本方视角列号”
- 横走用 `平`
- 直走或斜走用 `进` / `退`
- `进/退` 后的数字：
  - 车、炮、兵、将/帅 用步数
  - 马、相/象、仕/士 用目标列号

### 5. 同类棋子消歧

当同类型棋子存在歧义时：

- 优先使用 `前` / `后`
- 当前后仍不足以表达时，本次实现不继续扩展为更复杂的 `前中后` 或 `一二三` 兵排序方案
- 首版范围内仅要求稳定支持最常见的两子冲突

定义口径：

- 从本方视角看，更靠前的一枚记为 `前`
- 更靠后的一枚记为 `后`

适用对象：

- 车、炮、马、兵/卒 等同类棋子冲突场景

### 6. 兵卒特殊说明

兵卒在同列冲突时也使用 `前` / `后`：

- `红前兵进一`
- `黑后卒进一`

本次不实现多兵同列时的完整专业棋谱扩展。

## 实现边界

本次只改“记谱文本生成逻辑”，不改：

- 棋盘规则
- AI
- 悔棋逻辑
- 历史列表结构

## 实现方式

建议把记谱判断集中在 `src/game/notation.js` 中，新增：

- 本方视角列号计算
- 同类棋子扫描
- `前/后` 消歧判断
- 不同棋种的 `进/退/平` 目标标识规则

`src/game/state.js` 继续在 `applyMove` 时调用记谱函数生成 `notation`。

## 测试范围

至少补这些测试：

- 无冲突常规记法
- 黑方也使用中文数字
- 双车同类冲突时生成 `前车` / `后车`
- 双炮同类冲突时生成 `前炮` / `后炮`
- 同列兵卒冲突时生成 `前兵` / `后卒`
- 马、相/象、仕/士 的 `进/退` 使用目标列号而非步数

## 非目标

本次不实现：

- `前中后` 三兵以上复杂消歧
- 完整职业棋谱全部边角规则
- 棋谱导出
- 历史回放
