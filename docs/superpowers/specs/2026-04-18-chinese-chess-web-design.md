# 中国象棋人机博弈网页版设计

## 目标

构建一个纯前端中国象棋单页应用，支持：

- 人机对战
- 初级、中级、高级三个难度
- 悔棋
- 走棋历史查看
- 本地直接运行，无需后端

## 运行形态

- 技术方案：原生 HTML + CSS + JavaScript
- 部署方式：静态页面
- 启动方式：直接打开 `index.html` 或使用任意本地静态服务器

选择原生前端是为了控制实现复杂度，避免引入构建链，同时保证交付物可直接运行。

## 界面结构

页面分为两栏：

- 左侧：棋盘区域
- 右侧：控制面板

控制面板包含：

- 对局状态显示
- 难度切换
- 重新开始按钮
- 悔棋按钮
- 走棋历史列表

棋盘交互规则：

- 点击己方棋子选中
- 高亮当前棋子的合法落点
- 点击合法位置完成走子
- 非法操作给出轻量提示，不弹复杂对话框

## 规则与数据模型

核心状态包括：

- 10x9 棋盘
- 当前行棋方
- 棋子对象
- 着法历史栈
- 棋局结果状态

规则层负责：

- 初始化标准棋局
- 生成合法走法
- 执行走子
- 吃子处理
- 将军与胜负判断
- 悔棋回滚

每一步历史记录保存：

- 起点
- 终点
- 行棋方
- 移动棋子
- 被吃棋子
- 走法文本
- 执行前后的关键状态

## AI 设计

AI 使用统一的传统搜索引擎：

- 合法走法生成
- 局面评估函数
- Minimax + Alpha-Beta 剪枝

难度差异不靠三套逻辑，而通过参数区分：

- 初级：较浅搜索 + 明显随机扰动 + 更高概率从前几名候选步中随机选
- 中级：中等搜索 + 少量随机扰动 + 更稳定的吃子和兑子判断
- 高级：较深搜索 + 很低随机扰动 + 尽量选择最优步

评估函数包含：

- 基础子力分
- 位置分
- 将帅安全
- 机动性
- 吃子收益

目标不是职业引擎强度，而是让三个难度的体感差异明显：

- 初级会出现可利用失误
- 中级具备基本对抗性
- 高级能稳定压制新手

## 悔棋与历史记录

悔棋默认回退一个完整回合：

- 玩家最近一步
- AI 最近一步

如果 AI 尚未走子，则只回退玩家当前一步。

走棋历史按回合显示：

- 第 1 回合：红方走法 / 黑方走法
- 第 2 回合：红方走法 / 黑方走法

历史面板只做查看，不支持点击回放，以控制首版复杂度。

## 错误处理

- 非法走子不修改状态
- AI 思考期间锁定棋盘输入
- 对局结束后禁止继续落子
- 任何状态更新都以单一局面源数据为准，避免 UI 与棋局状态分离

## 测试范围

首版验证重点：

- 各棋种合法走法
- 将军/胜负判断
- AI 能正常完成思考与落子
- 三档难度行为差异可见
- 悔棋可稳定恢复棋局
- 历史记录与实际走子一致

## 范围边界

首版不包含：

- 联机对战
- 棋谱导入导出
- AI 对 AI
- 悔棋到任意历史步
- 开局库
- 残局模式
- 动画音效自定义

## 交付物

计划交付：

- `index.html`
- `styles.css`
- `app.js`

其中 `app.js` 内部按模块职责拆分为：

- UI 层
- 棋局规则层
- AI 搜索层
- 历史与悔棋层
